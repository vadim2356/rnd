# Проверка после SEO-улучшений

## 1. Итоговый `<title>` (без шаблона, 50–60 символов)

В `app/layout.tsx` убран `title.template`; каждая страница задаёт полный title через `generateMetadata()` или `metadata`. Ограничение в `buildMeta()`: 58 символов (обрезка до 55 + «…»).

**Примеры итогового HTML `<title>`:**

| Страница | Пример title | Символов |
|----------|---------------|----------|
| Услуга | Умягчение воды в Ростове-на-Дону — под ключ | 48 |
| Город + услуга | Обезжелезивание воды в Батайске — под ключ | 41 |
| Решение | Очистка воды из скважины в Ростове-на-Дону — под ключ | 52 |

Для городских страниц используется `namePrepositional` (в Ростове-на-Дону, в Батайске, в Аксае и т.д.), поэтому грамматика в title корректна.

## 2. Пример город-хаба с новыми блоками

На странице `/[citySlug]` (например, `/bataysk`) добавлены:

- **«Особенности воды в {город}»** — текст из `getCityWaterFeatures(cityName, citySlug)` (4 варианта, выбор по seed).
- **«Как мы работаем в {город}»** — текст из `getCityHowWeWork(cityName, citySlug)` (4 варианта).
- **FAQ** расширен до 6–8 вопросов (раньше 5–6); добавлены вопросы про юрлиц и обслуживание.

Объём страницы город-хаба: ориентировочно 700–900 слов (лид + услуги + особенности воды + как работаем + проблемы + кейсы + FAQ).

## 4. Сборка

Рекомендуется выполнить в каталоге проекта:

```bash
npm run build
```

Если в среде есть проблемы с путями с кириллицей (например, в PowerShell), запускайте сборку из каталога без кириллицы в пути (например, скопировав проект в `C:\Land`) или из WSL/другого терминала.

После успешной сборки все страницы, включая новые `/prices` и `/{citySlug}/prices`, будут сгенерированы статически (SSG).
